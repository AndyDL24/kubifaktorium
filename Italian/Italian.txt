#Il formato generale del file di localizzazione per ogni riga è <key> = <valore>, ad esempio "UI.BuildTrain = Build train". <key> non deve essere modificato, <value> deve essere tradotto, quindi ad es. "UI.BuildTrain = Baue Zug".
#Una riga che inizia con "#" viene ignorata e può essere utilizzata per i commenti.
#La riga "fallback = English" specifica quale lingua utilizzare quando non è possibile trovare una determinata stringa. Di solito l'inglese è una buona scelta in quanto è di solito la più aggiornata ma altre lingue possono essere scelte se sono più vicine alla lingua di destinazione.
#Il gioco supporta la sillabazione per le lingue con stringhe lunghe come il tedesco. Per usarlo, usa il carattere "_" per inserire un trattino morbido nei punti in cui una parola può essere sillabata. Lo consiglio per parole più lunghe di 12-15 caratteri. Ad esempio: "recommen_dations" o "trans_portation".
#Una stringa nel formato "# <id> #" provoca l'inserimento di un'icona, ad es. "#sole#". Si noti che l'id non deve essere tradotto, altrimenti non verrà trovato e tradotto correttamente.
#Una stringa nel formato "## <id>" crea una parola chiave. Parole chiave come "## daytime" verranno tradotte in una parola sottolineata "daytime". Inoltre, una spiegazione verrà aggiunta sotto, di solito nelle descrizioni comandi. Ancora una volta, l'id non deve essere tradotto. Tuttavia, il testo esplicativo (le righe con i tasti come ## inattivo) dovrebbe.
#Stringhe come "$ 1", "$ 2" ecc. Sono segnaposto per numeri e altre cose che verranno riempite dal gioco. Ad esempio "Tasso di crescita durante il giorno: $ 1%" diventerà "Tasso di crescita durante il giorno: 75%" durante il runtime.
#Se richiesto per qualche motivo, puoi utilizzare <br> per creare un'interruzione di riga. Comunque consiglio di non usarlo.
#Ogni volta che avvii il gioco con un nuovo file di traduzione, verrà creato automaticamente un elenco delle righe di traduzione mancanti. Ciò è utile sia per iniziare che per mantenere la traduzione aggiornata quando vengono aggiunte nuove stringhe. Il file può essere trovato in C: \ Users \ <nomeutente> \ AppData \ LocalLow \ MirkoStudio \ Kubifaktorium \ Missing_ <language> .txt su Windows o ~ / .config / unity3d / MirkoStudio / Kubifaktorium / su Linux.
